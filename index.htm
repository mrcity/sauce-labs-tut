To really get the most out of Sauce, though,
we need to explore the Selenium WebDriver.
The Selenium WebDriver presents you with a standard API to run identical commands across many browsers*.
* Selenium WebDriver has to be implemented for each browser, so it's not available for all browsers.
However, Selenium RC (which has been supported in Perl for many, many years) generally works with most anything.
Nevertheless, the Selenium WebDriver API tends to have less confusing code than Selenium RC API, so we'll use WebDriver here.
Start by downloading Selenium::Remote ::Driver from CPAN.

Haha, Fooled You!!
It's unlikely your localhost is exposed to the real world, so of course Sauce's VMs can't get to it.
To resolve this, Sauce Labs created:
Sauce Connect
Sauce Connect is a command-line tool that starts Selenium WebDriver and makes a tunnel for you so
you don't have to fumble around with exposing your precious localhost to the mean ol' World Wide Web
nor poke holes in your corporate firewall.
Another beauty of Sauce Connect is multiple sub-accounts can share a tunnel,
thus one (or very few) machines in the enterprise actually need to run Sauce Connect.
So let's start Sauce Connect right now and see how our fate changes!
At a minimum, you need to run sc -u your-username -k your-api-key
